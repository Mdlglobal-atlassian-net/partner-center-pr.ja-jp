# <a name="transition-from-cloud-solution-provider-csp-direct-bill-partner-to-csp-indirect-reseller"></a>クラウド ソリューション プロバイダー (CSP) 直接請求パートナーから CSP 間接リセラーへの移行 

>[!Note] 
>この記事では、間接リセラーへ移行することにしました直接請求パートナー向けです。 

直接請求の既存のテナントを使用して、間接リセラー プログラムに登録することができます。 

## <a name="get-started"></a>使ってみる

1. パートナー プロファイル パートナー センターでと確認 MPN ID が最新します。

2. 間接リセラーへの移行は直接請求テナントの全体管理者としてパートナー センターにサインインします。

![概要](images/direct/direct1.png) 

3. 登録フォームで、パートナーの詳細を確認します。

![今すぐ登録します。](images/direct/direct2a.png) 

4. [今すぐ登録する] を選びます。 間接リセラー ビジネスに直接ビジネスを使用する同じ AAD テナントが使用されます。

>[!Note]
>最初に、この新しい移行機能は、年 12 月締日に年 9 月にパートナーできるようになります。 表示されない"t 年 9 月、年 12 月の間、締日、この時点で、機能は表示されません。 2018 年 12 月後の締日でパートナーは、パートナーの機能が有効にすると、後で通知されます。


5. 登録が承認されると、パートナー センターにもう一度サインインします。

>[!Note]
> 承認は、通常は即時ですが、最大 5 営業日がかかることができます。 承認されると、登録フォームに第一連絡先担当者の下に指定したメール アドレスに通知を受け取ります。 **設定**→**パートナー設定**→**パートナー プロファイル**→ プログラム情報の下の登録状態をチェックすることもできます。 

6. **概要**ページで、間接リセラー契約が表示されます。 **[同意して続行する]** を選択します。 これにより、間接リセラーの機能です。

間接リセラー契約が受け入れられると、いるパートナー プロファイルを識別する**両方**として直接請求と間接リセラーに注目してください。

![間接リセラー契約](images/direct/direct3.png)

>[!Important] 
>新しい機能を使用して、間接リセラーとして登録すると、直接請求書にのみテナントにロールバックするオプションはありません。 間接リセラーとして登録する前に、ビジネス ニーズを完全に評価することを確認してください。

## <a name="while-you-transition-from-direct-to-indirect-reseller"></a>ながら間接リセラーに直接から移行します。

管理は引き続きこのフェーズでは、直接、顧客のサブスクリプションは、請求のプロセスを含む必要があります。 間接プロバイダーと間接リセラーとして運用から顧客を受け入れを開始することもできます。

![直接請求と間接リセラーの両方があります。](images/direct/direct4.png)

## <a name="find-an-indirect-provider"></a>間接プロバイダーを検索します。

左側のナビゲーションを登録した後の間接プロバイダーへのリンクが表示されます。 間接リセラーとしてし、処理、請求、お客様の製品を購入およびユーザー インフラストラクチャをサポート間接プロバイダーと関係が確立されます。

提供されるサポートやサービスは間接プロバイダーによって異なるため、リセラーは対象地域の複数のプロバイダーを評価して、自社のニーズに最も適したプロバイダーを判断する必要があります。 一般に、ほとんどのプロバイダーが提供している機能は次のとおりです。 

- リセラーへのテクニカル サポートの提供
- リセラーの製品やサービスのマーケティング支援 
- 調達や融資に関する条項を管理します。 

正式な[Microsoft 間接プロバイダー](https://partnercenter.microsoft.com/partner/find-a-provider)の一覧を検索します。

詳細については、[間接プロバイダーとパートナー](indirect-reseller-tasks-in-partner-center.md)を読み取る

## <a name="accept-a-partnership-invitation-from-your-indirect-provider"></a>間接プロバイダーから、パートナーシップへの招待を受け入れる

間接プロバイダーとの連携を検索するときは、パートナー センターで間接プロバイダーとパートナーシップを確立します。

選択した間接プロバイダーが送信メールでパートナーシップへの招待リンクをクリックすると、パートナー センターで、招待します。 必ず、グローバル管理者がパートナー センターにサインインし、招待リンクに依存してください。 招待を承諾すると、プロバイダーの名前は、間接プロバイダーの一覧に表示されます。

## <a name="acquire-new-customers-as-indirect-reseller"></a>間接リセラーとして新しい顧客を獲得します。

間接プロバイダーと顧客と再販業者関係が必要です。 これにより、顧客のサブスクリプションとサービスを代理で管理できます。 既存の Azure AD テナントを持つ新しい顧客を取得するには、同時に、貴社とプロバイダーの両方と再販業者関係を確立する顧客を招待することができます。 

間接リセラーの招待を作成します。

1. **間接プロバイダー**を選択して、パートナー センターの左側のナビゲーションから 

2. 貴社と間接プロバイダーの両方について再販業者関係を確立するために新しい顧客を招待するには、**[新しい顧客を招待する]** を選びます。 プロバイダーは、顧客が新しいサブスクリプションを購入または既存のサブスクリプションの新しいライセンスを追加するときに、顧客に代わって注文を送信できるように、顧客と再販業者関係がある必要があります。

3. 次のページで、メール メッセージの下書きを確認します。 メールで下書きメッセージを開くまたはメッセージをクリップボードにコピーしてメールに貼り付けます。

4. 何する必要がありますが自分のアカウントとプロバイダーのアカウントに直接、顧客を接続するカスタマイズされていますが、リンクを必ず指定するメールのテキストを編集します。 **終了**して選択します。

5. 顧客が貴社とプロバイダーを指名リセラーとして承認すると、貴社には顧客のサブスクリプション、ライセンス、ユーザーを管理するための管理者のアクセス許可が付与され、間接プロバイダーは、顧客に代わって注文を送信できるようになります。
6. 顧客のアカウント、サービス、ユーザー、ライセンスを管理するには、顧客の名前の下矢印を選んで、顧客のレコードを展開します。 

直接請求パートナーとは異なり間接リセラーは、パートナー センターで、新しい顧客の Azure AD テナントを作成することはできません。 プロバイダーは、テナントを作成し、この顧客に対して、間接リセラーとして指定します。 これにより、お客様がパートナー センターでの顧客一覧に表示されます。

>[!Note]
>間接リセラーとして取得したお客様の購入を作成する、直接請求機能を使用することはできません。

## <a name="move-your-direct-bill-customers-to-your-indirect-provider"></a>直接請求のお客様を間接プロバイダーに移動します。

再販業者関係があるまで、間接プロバイダーは、既存の直接請求顧客の注文を送信できません。 既存の顧客と間接プロバイダーの間で再販業者関係を確立するには、間接リセラーの招待を使用して顧客を招待します。 

1. **間接プロバイダー**を選択して、パートナー センターの左側のナビゲーションから

2. **新しい顧客を招待**を同時に、間接プロバイダーの両方と再販業者関係を確立する顧客を招待するかを選択します。 プロバイダーは、顧客が新しいサブスクリプションを購入または既存のサブスクリプションの新しいライセンスを追加するときに、顧客に代わって注文を送信できるように、顧客と再販業者関係がある必要があります。

3. 次のページで、メール メッセージの下書きを確認します。 メールで下書きメッセージを開くまたはメッセージをクリップボードにコピーしてメールに貼り付けます。 

4. 何する必要がありますが自分のアカウントとプロバイダーのアカウントに直接、顧客を接続するカスタマイズされていますが、リンクを必ず指定するメールのテキストを編集します。 **終了**して選択します。  

5. 顧客が貴社とプロバイダーを指名リセラーとして承認すると、貴社には顧客のサブスクリプション、ライセンス、ユーザーを管理するための管理者のアクセス許可が付与され、間接プロバイダーは、顧客に代わって注文を送信できるようになります。

6. 顧客のアカウント、サービス、ユーザー、ライセンスを管理するには、顧客の名前の下矢印を選んで、顧客のレコードを展開します。

>[!Note]
>現時点では、ビジネス向け Microsoft ストアでは、既存の直接請求パートナーから間接リセラーの招待を承諾する顧客を許可しません。 ビジネス向け Microsoft ストアをサポートしてこの、近い将来にします。 それまでは、引き続き、直接請求機能を使用して既存の直接請求顧客の注文を送信します。

## <a name="managing-your-direct-bill-customers-and-your-indirect-reseller-customers"></a>直接請求のお客様と、間接リセラーの顧客を管理します。

直接請求のお客様と、間接リセラーの顧客を異なる方法で管理します。 

**直接請求書ユーザー (操作を間接リセラーとして実行されません)**

- 製品の注文を作成します。
- Azure reservations を管理します。
- 自分の注文履歴を管理します。
- ソフトウェアを購入します。
- ユーザーを直接請求します。

**間接リセラーの顧客**

- 間接プロバイダーが顧客の製品を注文します。
- 顧客のライセンスとユーザーを管理します。
- サブスクリプションの更新を処理します。

**直接請求パートナーとして入手したユーザーを識別するには**

1. **顧客**を選択します。

2. その詳細を表示する顧客を選択します。

3. このユーザーが直接請求パートナーとして取得する場合は、オプションを**追加する**か、**製品の表示**が表示され、サブスクリプションが表示されます。

4. お客様が場合、間接リセラー関係すると、これらのオプションは利用できません。

## <a name="enroll-for-indirect-reseller-incentives"></a>間接リセラーのインセンティブ登録します。 

正常に、既存の直接請求パートナー テナントで間接リセラーとして登録した場合後から 30 日以内の間接リセラーのインセンティブ登録の招待を受け取ります。 招待は、現在、CSP パートナーのテナントに関連付けられている MPN アカウント パートナーに基づいています。 招待は、パートナーの MPN アカウントに関連付けられているメール アドレスに送信されます。 

同じパートナー テナントで直接請求インセンティブ プログラムの登録の対象とする必要があります。 プログラムを個別に管理する必要があります。
