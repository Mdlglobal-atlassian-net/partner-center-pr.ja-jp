# <a name="transition-from-cloud-solution-provider-csp-direct-bill-partner-to-csp-indirect-reseller"></a>クラウド ソリューション プロバイダー (CSP) 直接請求パートナーから CSP 間接リセラーへの移行 

>[!Note] 
>この記事では、直接請求書のパートナーが間接リセラーに遷移することが決定されました。 

既存の直接の請求書テナントを使用して間接リセラー プログラムに登録できるようにします。 

## <a name="get-started"></a>作業開始

1. 確認、パートナーのプロファイルでは、パートナー センターと MPN ID は現在します。

2. 間接リセラーに移行している請求書の直接のテナントのグローバル管理者としてパートナー センターにサインインします。

![概要](images/direct/direct1.png) 

3. 登録フォームに、パートナーの詳細を確認します。

![今すぐ登録します。](images/direct/direct2a.png) 

4. 選択は、今すぐ登録します。 間接リセラー ビジネスでは、直接ビジネスを使用する同じ AAD テナントを使用します。

>[!Note]
>最初に、この新しい移行機能は、年 12 月の日に年 9 月とパートナー向けになります。 "T 記念日 9 月と年 12 月の間、この時点で、機能は表示されませんないかどうか。 2018 年 12 月日より後の記念日でパートナーは、パートナーの機能が有効にすると、後で通知されます。


5. お客様の加入契約が承認されると、パートナー センターにもう一度サインインします。

>[!Note]
> 承認は、通常は即時ですが、最大 5 営業日かかります。 承認されると、登録フォームの主要な連絡先下に指定した電子メール アドレスに通知が届きます。 下で、登録状態を確認することもできます。**設定**→**パートナー設定**→**パートナー プロファイル**] → [番組情報。 

6. **概要** ページで、間接リセラー契約が表示されます。 **[Accept and continue] (同意して続ける)** を選びます。 これにより、間接リセラーの機能です。

間接リセラー契約が受け入れられると、確認として、パートナーのプロファイルを識別する**両方**請求書を直接と間接リセラーをします。

![間接リセラー契約](images/direct/direct3.png)

>[!Important] 
>新しい機能を使用して間接リセラーとして登録すると、直接請求書にのみテナントにロールバックするオプションはありません。 間接リセラーとして登録する前に、ビジネス ニーズを完全に評価することを確認してください。

## <a name="while-you-transition-from-direct-to-indirect-reseller"></a>直接間接リセラーに移行するときに

このフェーズで、引き続き管理することは、ダイレクトのお客様のサブスクリプションは、請求プロセスを含む必要があります。 間接プロバイダーと間接リセラーとして動作している顧客の受け入れを開始することもできます。

## <a name="find-an-indirect-provider"></a>間接プロバイダーの検索

左側のナビゲーション、登録後に間接プロバイダーへのリンクが表示されます。 間接リセラーとしてできますの課金情報の処理、お客様の製品を購入し、インフラストラクチャをサポート、間接プロバイダーと関係が確立されます。

提供されるサポートやサービスは間接プロバイダーによって異なるため、リセラーは対象地域の複数のプロバイダーを評価して、自社のニーズに最も適したプロバイダーを判断する必要があります。 一般に、ほとんどのプロバイダーが提供している機能は次のとおりです。 

- リセラーへのテクニカル サポートの提供
- リセラーの製品やサービスのマーケティング支援

## <a name="accept-a-partnership-invitation-from-your-indirect-provider"></a>間接プロバイダーからパートナーシップへの招待を受け入れる

提携のため、間接プロバイダーが見つかったら、パートナー センターで間接プロバイダーとのパートナーシップを確立します。

選択した間接プロバイダー送り電子メールでパートナーシップ招待リンクをクリックすると、パートナー センターで、招待します。 グローバル管理者は、パートナー センターにサインインして、招待のリンクに従うことを確認します。 招待を承諾すると、プロバイダーの名前は、間接プロバイダーの一覧に表示されます。

## <a name="acquire-new-customers-as-indirect-reseller"></a>間接リセラーとして新しい顧客を獲得します。

間接プロバイダーとは、顧客と販売店のリレーションシップを持つ必要があります。 これにより、お客様のサブスクリプションと、ユーザーに代わってサービスを管理することができます。 既存の Azure AD テナントを持つ新しい顧客を取得するためには、同時に、プロバイダーと再販業者関係を確立するために顧客を招待できます。 

間接リセラーの招待を作成します。

1. 選択**間接プロバイダー**パートナー センターの左のナビゲーション 

2. 貴社と間接プロバイダーの両方について再販業者関係を確立するために新しい顧客を招待するには、**[新しい顧客を招待する]** を選びます。 プロバイダーは、お客様が新しいサブスクリプションを購入するか、既存のサブスクリプションに新しいライセンスを追加するときに、お客様の代わりに注文を送信できないように、お客様と再販業者関係を持つ必要があります。

3. 次のページで、メール メッセージの下書きを確認します。 電子メールで、ドラフトのメッセージを開くか、またはメッセージをクリップボードにコピーして電子メールに貼り付けます。

4. 必要がある確認が、パーソナル化アカウントと、プロバイダーのアカウントの両方に直接、顧客を接続するように、リンクを含めるようにするくださいと電子メールのテキストを編集します。 選び**完了**します。

5. 顧客が貴社とプロバイダーを指名リセラーとして承認すると、貴社には顧客のサブスクリプション、ライセンス、ユーザーを管理するための管理者のアクセス許可が付与され、間接プロバイダーは、顧客に代わって注文を送信できるようになります。
6. 顧客のアカウント、サービス、ユーザー、ライセンスを管理するには、顧客の名前の下矢印を選んで、顧客のレコードを展開します。 

直接の請求書のパートナーとは異なり、間接リセラーはパートナー センターで、新しい顧客の Azure AD テナントを作成できません。 プロバイダーは、テナントが作成され、この顧客の間接リセラーとして指定します。 これにより、顧客がパートナー センター顧客リストに表示されます。

>[!Note]
>直接の請求書の機能を使用して、間接リセラーとして取得する顧客の購入を作成することはできません。

## <a name="managing-your-direct-bill-customers-and-your-indirect-reseller-customers"></a>お客様の請求書を直接と間接リセラーの顧客を管理します。

請求書が直接お客様と、間接リセラーの顧客を異なる方法で管理します。 

**ダイレクトの請求書お客様 (間接リセラーとして実行しないこと)**

- 製品の注文を作成します。
- Azure 予約を管理します。
- 自分の注文履歴を管理します。
- ソフトウェアを購入します。
- 顧客に直接課金します。

**間接リセラーの顧客**

- 間接プロバイダーは、お客様の製品を注文します。
- 顧客のライセンスとユーザーを管理します。
- サブスクリプションの更新を処理します。

**請求書の直接パートナーとして取得した顧客を識別するには**

1. 選択**顧客**

2. その詳細を表示する顧客を選択します。

3. この顧客が 1 つのパートナーは直接請求書を入手した場合は、オプションが表示されます**追加**または**製品の表示**し、そのサブスクリプションが表示されます。

4. お客様に間接リセラーの関係をした場合、これらのオプションは利用できません。

## <a name="affected-direct-bill-capabilities"></a>直接の請求書の機能に影響を受ける

直接請求書の機能が制限されている、ときに、パートナー センターで顧客の新規購入できなく作成できます。 たとえば、次のようなアニメーションや効果を作成できます。 

- Azure サブスクリプション
- ライセンス ベースのサブスクリプション
- 既存のライセンス ベースのサブスクリプションへの新しいアドオンの追加
- サブスクリプションのソフトウェア、永続的なソフトウェアは、Azure 予約仮想マシン インスタンスなどのソフトウェアと予約の製品の 1 回限りの購入を行います。

CSP プログラムでの Azure パートナー共有サービス プランを使用して、独自の使用の新しい Azure サブスクリプションを購入することはできませんもします。 

だけでなく新規購入、パートナー センターでは、次の直接請求機能にアクセスすることはできません。 
- 新しい顧客のテナントを作成することはできません。 **顧客の作成**オプションで **顧客**パートナー センターでは使用できません。 

- 直接再販業者関係を要求するユーザーに招待状を生成することはできません。 **再販業者関係の要求**オプションで **顧客**パートナー センターでは使用できません。 

>[!Note] 
>間接リセラーとして、直接課金パートナーのテナントを既に登録した場合に、直接請求書ではなく、間接リセラーの関係を要求するユーザーに招待状を送信することができます。 

- サンド ボックスに新しいテナントを作成することはできません。 各請求書の直接パートナーのテナントでは、直接請求 API の統合のために 1 つのサンド ボックス テナントを作成できます。 1 つは既に作成していない場合は、ことできません課金パートナーの機能が制限されている指示するための処理方法。 

## <a name="whats-not-affected"></a>どのような影響はありません。

既存のサブスクリプションの請求書を直接は影響しません。 有効なままされ、自動更新されます。 キャンセルするまで、Microsoft によって直接課金される続けます。 

続行すると、次の方法で既存のサブスクリプションを管理します。 

- 既存のサブスクリプションを中断します。 

- 既存のサブスクリプションのライセンス ベースの接続クライアント数を調整します。 

- サブスクリプションにアドオンを既存の接続クライアント数を調整します。 注:アドオンが新規購入として扱われるために、既存のサブスクリプションに新しいアドオンを追加することはできません。 

- 新しい Azure リソースをデプロイし、既存の Azure サブスクリプションの下で、既存の Azure リソースを管理します。 これには、Azure marketplace と Visual Studio サブスクリプションを通じて使用可能な製品が含まれます。 

## <a name="enroll-for-indirect-reseller-incentives"></a>間接リセラー インセンティブを登録します。 

既存の直接の請求書パートナー テナントでは、間接リセラーとして正常に登録が後から 30 日以内に間接リセラー インセンティブを登録する招待を受け取ります。 招待は、パートナー、CSP パートナーのテナントに現在関連付けられている MPN アカウントに基づいています。 招待は、パートナー MPN アカウントに関連付けられた電子メール アドレスに送信されます。 

その同じパートナー テナントを使用して直接課金インセンティブ プログラムを登録する対象となる必要があります。 プログラムを個別に管理する必要があります。

参照してください[間接リセラーになるために追加情報](https://assetsprod.microsoft.com/csp-directbill-to-indirect-transition.pdf)と[間接リセラーに直接請求に関する fAQ](https://assetsprod.microsoft.com/mpn/direct-bill-partner-faq.pdf)します。
