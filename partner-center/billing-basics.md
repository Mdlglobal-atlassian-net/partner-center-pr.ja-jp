# <a name="billing-overview"></a>課金の概要

**適用対象**

-  パートナー センター
-  CSP プログラムのパートナー

によって、製品、ソリューション、およびお客様の代わりに購入したサービスは、課金されるで、次の方法の 1 つ以上のこれらの購入の。
-   [ライセンス ベースの課金](#licensebasedbilling)

    製品またはライセンスを必要とするオンライン サービスを購入するときに、(ライセンスの使用状況) ではなく購入したライセンスごとに請求されます。 課金は月に 1 回または 1 年間に 1 回かどうかを選択できます。 ビジネス ニーズの変化場合、他のいずれかから切り替えるし、もう一度バックアップできます。 
    
    年次請求と月単位の詳細については、請求先住所を参照してください。 [FAQ](https://docs.microsoft.com/en-us/partner-center/faq-about-new-billing-features)します。

-   [使用量ベース課金](#usagebasedbilling)

    Azure サブスクリプションなどのオンライン サービスを購入するときに、月間使用料金の請求されます。 のみ月額使用料は、製品の使用法に基づく利用できます。 Azure などの使用量ベースのサービスは、消費量に基づいて、従量制課金の料金に従って課金されます。

-   [1 回限りの課金](#onetimebilling)

    Azure 予約またはその他のソフトウェア サブスクリプションを購入するときに、事前設定された用語の事前に課金します。 事前に、料金を支払っているため、一括の 1 つが課金されます。 
    
月ごとに課金することを選択した場合、または月ごとに課金される使用量ベースの製品を購入した場合は、毎月、ご請求日は、パートナー センターで、CSP アカウントの作成時に選択した月の日です。 CSP アカウントを正常に作成した後、Microsoft は、ご請求日を含む確認の電子メールを送信します。 作成されると、この日付は変更できません。 

## <a name="pricing-and-invoicing"></a>価格と請求
価格表は毎月更新され、1 か月前に確認することができます。 ライセンスベースの価格は、サブスクリプションの有効期間 (通常は購入日から 12 か月間) にわたって保証されます。 使用量ベースの価格は、毎月変わる場合があります。 

更新すると、価格を変更することがありますが、サブスクリプション期間だけで、製品、サービス、およびソフトウェア サブスクリプションの価格が保証されます。

クレジットまたは調整が適用されると、次回の請求書に未払いの調整とクレジットが表示されます。

請求書と調整ファイルは、パートナー センターの [請求書作成] ページで表示およびダウンロードできます。 パートナー センターでは、選択した請求日の 4 日以内であれば、毎月の請求書を確認できます。

## <a name="payment-terms"></a>支払条件

支払条件は、net の 60 日間です。 請求書で支払う必要がある期限 (請求期限後は 60 日) または、アカウント、請求書になります、滞納 in CSP のお客様の加入契約に影響する可能性があります。 中断されているアカウントのすべての機能を復旧するには、過去に支払うときに期限の量。

### <a name="tax"></a>Tax

ベース (いないお客様)、詳細酷使がは請求関係は、Microsoft との間は。 アカウントのセットアップ プロセス中に、または後で、サポート リクエストを送信することで、税 ID を送信できます。 料金は次の課金サイクルに反映されます。

-   源泉徴収と税抜き除外は、サポート要求を通じて税のドキュメントを送信する必要があります。 料金と適切な返金は次の課金サイクルに反映されます。

-   付加価値税 (VAT) 除外対象のサービス要求を使用して (Microsoft によって検証)、VAT ID を送信する必要があります。 料金と適切な返金は次の課金サイクルに反映されます。

## <a href="" id="licensebasedbilling"></a>ライセンス ベースの課金

顧客の代理としてライセンス ベースの製品を購入するときに、課金は月単位または年 1 回選択できます。 後で、請求頻度を変更する場合は、以下の手順を使用します。 

月ごとに課金は多数のサブスクリプションがあるし、し、一般的な請求日に調整する場合は、毎月の課金から年次請求に切り替えると便利です。 年次請求から毎月の課金への切り替えは、各顧客の請求、日付を調整することに役立ちます。 

請求頻度を変更すると、年間契約は、請求頻度を変更した日付と新しい更新日が確立されているを反映するように更新されます。 

ビジネス ニーズの変化するたびに、請求頻度を変更できます。 

### <a name="billing-rules-for-annual-billing"></a>年次請求請求ルール

-   サブスクリプションは毎年自動更新されます。

-   請求は、年間サブスクリプションごとに 12 回の月払いまたは 1 回の年払いになります。

-   前回の請求期間の終了時のライセンス数に基づき、ライセンスベースのサービスの次の請求期間分が前払いで請求されます。

-   ライセンス数の変更 (ライセンスの日数に基づく日割り計算) で、後で課金/クレジットが行われます。 日割り計算では、次の式を使用します。[ROUND((ROUND(単価 * 数量 / 日割り計算の月の日数, 2) * 日割り計算の日数) / 数量, 2) * 数量]

-   支払いは、販売 (ライセンスはプロビジョニングされません) のライセンスに対して課金されます。

### <a name="to-change-the-billing-frequency-of-an-online-service"></a>オンライン サービスの請求頻度を変更するには

1.  選択**顧客**を選択し、パートナー センターのメニューから顧客サブスクリプションに変更したいです。 

2.  お客様のサブスクリプション ページで、変更するサブスクリプションを選択します。 

3.  詳細 ページで、**請求頻度**を選択します**毎月**または**年間**します。 変更されようとしてサブスクリプションの一覧と同様に、請求頻度を変更する方法についての重要な情報の確認ページが表示されます。 

4.  選択**OK**に変更を行い、または**キャンセル**元に戻すこと。 

### <a name="adjustmentscreditscancellations"></a>調整/クレジット/取り消し

Microsoft では、ライセンス ベースのサービスのキャンセルの中途解約料は料金がかかりません。 

ライセンス ベースのサービスに対する取り消しのクレジットは、期間途中の取り消しについては未使用の日について日割り計算されます (前の式に従ったライセンスの減少も同様です)。

## <a href="" id="usagebasedbilling"></a>使用量ベース課金

一部の Microsoft 製品とサービスは、"従量課金制"の課金モデルを使用されるサービスに対してのみ課金されますを使用します。 たとえば、Microsoft Azure は、このモデルを使用します。 

### <a name="billing-rules"></a>課金の規則
-   サブスクリプションは月ごとに課金され、新しい従量制課金サービス料金を自動的に更新します。 前月の使用量を毎月課金されます。

-   従量制サービスの料金は、請求サイクル内で変更される場合があります。 

    -   価格は増加します。30 日間通知が提供されます。

    -   価格の低下: 変更日を反映します。

    -   既存のサブスクリプションでは、課金サイクルの開始時点で有効な料金が使われます。

    -   同じの請求サイクル内で作成されたときに、新しいサブスクリプションは、作成日に有効で、レートを使用します。 

### <a name="adjustmentscreditscancellations"></a>調整/クレジット/取り消し

支払いの調整は、次の月次請求書に表示されます。

Microsoft では、使用法に基づくサービスのキャンセルの中途解約料は料金がかかりません。 

SLA のクレジットを含むあらゆる種類のクレジットが、次の月次請求書に表示されます。

## <a href="" id="onetimebilling"></a>1 回限りの課金

1 年または 3 つの条件のソフトウェア サブスクリプションと Azure の予約を事前に購入できます。 事前に購入するときに、1 つ一括で全体のコストが請求されます。 この種類の課金は、1 回限りの課金と呼ばれます。

>[!IMPORTANT]
>Azure 予約や異なる通貨を使用する位置での顧客ソフトウェア サブスクリプションを購入する場合、既定値は通貨の課金は、場所ではなく、顧客の場所に基づきます。 顧客が複数の場所にある、独立した請求書が届きます各通貨の顧客の調整ファイル必要がある場合に課金されます、適切な通貨でのお客様の請求可能です。 

### <a name="manage-your-one-time-billing"></a>1 回限りの請求を管理する

**および課金の現在の状態や請求書、recon ファイルを表示します。**

2.  請求書または調整ファイルを選択して、詳細な情報を確認します。 

**顧客の注文履歴を表示します。**

1.  選択**顧客**パートナー センターのメニューから。

2.  **[顧客]** ページで、注文履歴を表示する対象の顧客を見つけ、下矢印を選択して顧客レコードを展開します。 

3.  **[View orders]** (注文の表示) を選択して注文履歴を表示します。

**貸方票をダウンロードします。**

クレジット_カードまたは rebill を要求する必要がある場合を取り上げます貸方票を元の請求書をキャンセルします。 次の理由でクレジット/rebill を要求する可能性があります。

-   請求先住所または発注書の訂正

-   請求書が生成された後、税金の払い戻しが適用された場合。 クレジット/再請求を要求して、元の請求書に税金の払い戻しを反映させることができます。 これは払い戻しの場合にも適用され、元の請求書のクレジット/再請求を要求し、払い戻しを受けることができます。
