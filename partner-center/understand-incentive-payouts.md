#<a name="understand-your-incentive-payouts"></a>インセンティブの支払いについて

**Applies to**

-  パートナー センター

インセンティブ管理者とユーザーは、インセンティブ プログラムから受け取った、またはまもなく受け取る収益と支払いの概要を確認することができます。   

##<a name="summary-information-on-earnings-and-payout"></a>Summary information on earnings and payout

インセンティブ ユーザーおよびインセンティブ管理者は、参加しているインセンティブ プログラムの資金状態を **[入金状況]** で確認できます。 必要に応じて、概要情報の各領域を詳しく分析できます。 

[概要] ページでは、支払いを受け取る月とプログラムの種類について全体的な傾向を分析して把握できます。 これに対して、獲得したインセンティブをプログラムごとや場所ごとに細分化して表示することもできます。 これにより、成功を収めている分野と、マーケティングと販売にさらに力を入れる必要がある分野を理解できます。 [入金状況] ページには、収益と支払いが表示されます。インセンティブ データは、48 時間ごとに更新されます。 グラフと表には最新の情報が反映されます。 

[概要] ページでは、次の 3 つの重要なインジケーターが使用されます。

|**重要なインジケーター**   |**説明**   |
|---------------------|:-----------------|
|今までの入金総額|マイクロソフトがその時点までにパートナー様に支払った累計金額。|
|未払い収益|収益を上げたが、まだ支払いがスケジュールされていない金額|
|今後の入金| 支払いがスケジュール済みの金額|


各インセンティブ プログラムの収益と支払いを分析します。 収益と支払いの間でタイムラインを追跡すると、リソースの今後のニーズなどを計画するのに役立ちます。 またインセンティブ プログラムで、さらに成功を追求し、積極的に取り組むべき領域も特定できます。 

## <a name="payment-trends"></a>お支払いの傾向

概要ページのお支払いの傾向のセクションでは、パートナー様が登録したプログラムごとに入金額が分割して表示されます。 お支払いの傾向ビューは、ページのデータ フィルターを使って制御できます。 お支払いスケジュールは、プログラムごとに定められています。 詳しくは、プログラム ガイドをご覧ください。 

## <a name="total-earnings-map"></a>合計収益マップ

合計収益マップは、パートナー様がどのロケールで収益を上げているかを示します。 取引を行った場所が、マップ上に反映されます。 たとえば、米国で販売すると、それがマップ上に反映されます。 このビューは、すべてのプログラムで共通です。 

## <a name="product-earnings-summary"></a>製品収益の概要

製品/プログラムの収益とお支払いの表には、収益合計額、未払い収益、今後の入金、現在までの支払い合計について、プログラムや製品ごとの内訳が表示されます。 



## <a name="analyze-incentives-earnings"></a>インセンティブの収益の分析

パートナー様の組織がインセンティブ プログラムからさらに収益を得る余地がある領域と、既に十分に収益を上げている領域を調べます。 This data makes it clear where you have earned but have not yet been paid.  Understand which programs and activities have been most successful as well as find out which of your locations is earning. By program you can see what the total earnings potential is, the time during which you can participate in the program and when it expires. プログラムと収益の種類ごとに、要件と開始日および終了日が異なります。 

## <a name="earning-trends"></a>収益の傾向 

収益の傾向のグラフは、収益の進展や変化の全般的な方向を示します。 このグラフ ビューは、日付範囲のセレクターで制御すると、選択した日付の範囲に応じてデータが表示されます。 

### <a name="earning-trends-category-definitions"></a>収益の傾向で使用するカテゴリの説明

|   |**説明**   |
|----------------|:--------------|
|支払い済み収益|入金済みの収益|
|未払い収益|支払いがまだスケジュールされていない収益|

### <a name="earning-types"></a>収益の種類

収益には、次の 2 つの種類があります。

- リベート
- 調整

## <a name="unpaid-earnings-summary"></a>未払い収益の概要

未払い収益の表には、支払いがスケジュールされていない未払い収益の概要が表示されます。 この表は、プログラム/製品と収益の種類ごとに表示されます。 収益期間の開始日と収益獲得が完了する終了日を確認できます。 収益有効日は、収益の支払期日が到来し、支払いを受ける資格が発生する日です。 収益に有効期限があり、それを過ぎると支払われない場合は、有効期限日が適用されます。

## <a name="payments-and-statements"></a>支払いと明細

支払いと明細のページのマップには、取引の支払いが場所に基づいて表示されます。 たとえば、米国で販売し、その支払いも米国で受け取った場合は、そのことがマップ上に表示されます。 このビューでは、参加したすべてのプログラムが集約されています。

## <a name="payments"></a>支払い

支払いの表には、過去 12 か月間に行われたすべての支払いが表示されます。 この表は、プログラム/製品と収益の種類ごとに表示されます。 収益期間の開始日と終了日を確認できます。 支払いが銀行に送金されると、支払日と支払い ID が表示されます。

## <a name="statements"></a>明細

支払いが銀行に送信されると、明細が生成されます。 **[表示]** をクリックすると、入金の詳細情報が表示されます。 

[取引のダウンロード] のリンクをクリックすると、その月に発生する入金に関連する取引と収益の詳細ビューが表示されます。 

支払いの詳細をダウンロードできるリンクをクリックすると、支払いの内訳を確認できます。 支払いが複数の支払いに分割された場合、この詳細ビューで分割方法を確認できます。
