# <a name="customize-a-devices-out-of-box-experience-with-windows-autopilot-profiles"></a>Windows Autopilot プロファイルを使用したデバイスの既定のエクスペリエンスをカスタマイズします。

**適用対象**

- CSP の請求書の直接パートナー、間接プロバイダー、および間接リセラー

お客様のデバイスを管理している場合は、顧客のユーザーの既定の experience (OOBE) をカスタマイズする必要があります。 デバイスを顧客に納入する前に Windows Autopilot プロファイルによって、新しいデバイスを事前に構成し、顧客が既に購入したデバイスに新しいプロファイルを適用できます。 この記事では、作成し、Autopilot プロファイルをパートナー センターでデバイスに適用する方法について説明します。

慣れていない既に Autopilot 場合のこれらの記事の情報を確認します。

- [Windows Autopilot の概要](https://docs.microsoft.com/windows/deployment/windows-10-auto-pilot)
- [Autopilot 展開リファレンス ガイド](https://assetsprod.microsoft.com/autopilot-deployment-program-reference-guide-csp.docx)  

## <a name="overview"></a>概要

パートナー センターで Windows Autopilot の機能、顧客のデバイスに適用するカスタム プロファイルを作成できます。 次のプロファイル設定は、この記事の公開時に使用可能なです。

- プライバシーの設定をスキップします。 省略可能なこの Autopilot プロファイルの設定を使用することにより、OOBE プロセス中に、プライバシーの設定について確認されません。

- デバイスのローカル管理者アカウントの作成を無効にします。 組織では、かどうか、プロセスが完了したら、デバイスを設定するユーザーが管理者アクセスする必要がありますを決定できます。

- 職場または学校のデバイスを自動的に設定します。 Autopilot に登録されたすべてのデバイスでは、作業が自動的に考慮するまたは学校のデバイスのため、この質問は、OOBE プロセス中には求められません。

- Cortana、OneDrive、および OEM の登録セットアップ ページをスキップします。 Autopilot に登録されたすべてのデバイスは、既製の experience (OOBE) プロセス中に自動的にこれらのページをスキップします。

- エンド ユーザー ライセンス契約 (EULA) をスキップします。 組織は Windows 10 バージョン 1709 以降、OOBE プロセス中に表示される使用許諾契約書 ページをスキップする決定できます。 参照してください[Windows Autopilot の使用許諾契約書ジョンソン](#windows-autopilot-eula-dismissal)以下の Windows の中に、EULA ページをスキップしています。 詳細については考慮すべき重要な情報をセットアップします。

次のプロファイルとデバイスの管理アクセス許可と制限事項が適用されます。

- CSP パートナーは顧客には、パートナーの管理の委任特権が削除された場合でも、再販業者関係のある相手の既存のお客様の Autopilot プロファイルを管理する続行できます。

- 追加した顧客の既存のデバイスを管理できます。

- お客様が Microsoft Store for Business または Microsoft Intune ポータルにアップロードするデバイスを管理することはできません。

## <a name="create-and-manage-autopilot-profiles-in-partner-center"></a>作成し、パートナー センターでの Autopilot プロファイルの管理

パートナー センターでは、Windows Autopilot deployment プロファイルを作成し、デバイスにも適用されます。

>[!NOTE]
>管理エージェントのみを作成し、プロファイルを適用できます。

### <a name="create-a-new-autopilot-profile"></a>新しい Autopilot プロファイルを作成します。

1. 選択**顧客**を選択し、パートナー センターのメニューから、顧客作成するの Autopilot プロファイル。

2. 顧客の詳細 ページで、次のように選択します。**デバイス**します。

3. **Windows Autopilot プロファイル**選択**新しいプロファイルを追加する**します。

4. プロファイルの名前と説明を入力し、OOBE 設定を構成します。 以下から選択します。  

   - セットアップでのプライバシー設定をスキップします。

   - セットアップでローカル管理者アカウントを無効にする
  
   - セットアップで自動的にページをスキップする<br>
        (が含まれています*職場または学校のセットアップが自動的に選択*と*Skip Cortana、OneDrive、および OEM の登録セットアップ ページ*)
  
   - エンド ユーザー ライセンス契約 (EULA) をスキップします。<br> 
       >[!IMPORTANT] 
       >参照してください[Windows Autopilot の使用許諾契約書ジョンソン](#windows-autopilot-eula-dismissal)以下の Windows の中に、EULA ページをスキップしています。 詳細については考慮すべき重要な情報をセットアップします。

5. 完了したら、**[送信]** を選びます。

### <a name="apply-an-autopilot-profile-to-customer-devices"></a>お客様のデバイスに、Autopilot プロファイルを適用します。

>[!NOTE]
>以下の手順では、パートナー センターに、お客様のデバイスを既に追加したことと、デバイスの一覧にアクセスできることを仮定します。 お客様のデバイスに追加済みしていない場合の指示に従って[、お客様のアカウントにデバイスを追加](#add-devices-to-a-customers-account)し、次の手順に従います。

Autopilot プロファイルを顧客を作成した後は、お客様のデバイスに適用できます。

1. 選択**顧客**を選択し、パートナー センターのメニューから、顧客プロファイルを作成した、Autopilot にします。

2. 顧客の詳細 ページで、次のように選択します。**デバイス**します。

3. **プロファイルをデバイスに適用**デバイスまたはデバイス グループのプロファイルを追加し、選択する選択**プロファイルを適用**します。 適用したプロファイルが表示されます、**プロファイル**列。

4. プロファイルをデバイスに正常に適用されることを確認するのには、次の手順に従います。

    a.   デバイスをネットワークに接続して、オンにします。

    b.   適切な OOBE 画面 (存在する場合) が表示されることを確認します。

    c.  OOBE プロセスが停止したら、新しいユーザーのための準備を出荷時の既定設定にデバイスをリセットします。

### <a name="remove-an-autopilot-profile-from-a-customers-device"></a>顧客のデバイスから Autopilot プロファイルを削除します。

1. 選択**顧客**を選択し、パートナー センターのメニューから、顧客プロファイルを作成した、Autopilot にします。

2. 顧客の詳細 ページで、次のように選択します。**デバイス**します。

3. **プロファイルをデバイスに適用**からプロファイルを削除しを選択するデバイスを選択します。**プロファイルを削除する**します。

   >[!NOTE]
   >デバイスからプロファイルを削除しても、一覧から、プロファイルは削除されません。 プロファイルを削除する場合は、次の手順に[Update または delete Autopilot プロファイルを](#update-or-delete-an-autopilot-profile)します。

### <a name="update-or-delete-an-autopilot-profile"></a>更新または削除、Autopilot プロファイル

顧客にデバイスを出荷した後は、既製のエクスペリエンスを変更する場合は、パートナー センターでプロファイルを変更できます。

顧客のデバイスは、インターネットに接続するとき、OOBE プロセス中に、最新のプロファイル バージョンがダウンロードされます。 また、顧客は、デバイスを工場出荷時の既定の設定に戻し、いつでも、デバイスはもう一度最新バージョンをダウンロード プロファイル OOBE プロセス中に。

1. 選択**顧客**から、パートナー センターのメニューとし、Autopilot プロファイルを変更する必要がある顧客を選択します。

2. 顧客の詳細 ページで、次のように選択します。**デバイス**します。

3. **Windows Autopilot プロファイル**を更新する必要があるプロファイルを選択します。 内容を変更し、**送信**します。

このプロファイルを削除するには、次のように選択します。**プロファイルを削除する**ページの右上隅にあるからです。

### <a name="add-devices-to-a-customers-account"></a>お客様のアカウントにデバイスを追加します。

>[!NOTE]
>販売エージェントと管理エージェントは、お客様のアカウントにデバイスを追加できます。

カスタムの Autopilot プロファイルをお客様のデバイスに適用する前に、お客様のデバイスの一覧にアクセスできる必要があります。

OEM 名、シリアル番号、およびモデルの組み合わせを使用する場合は、これらの制限の注意。

- この組は、以降のデバイス (4 k ハッシュなど) に対してのみ 128b ハッシュ (RS2 と以前のデバイス) はサポートされていません。

- ファイル内のデータは、モデルと製造元の名前と一致する必要がありますので、タプルの登録は大文字小文字が区別***まったく***OEM プロバイダー (ハードウェア プロバイダー) によって提供されます。

パートナー センターでお客様のアカウントにデバイスを追加するには、以下の手順に従います。

1. 選択**顧客**パートナー センターのメニュー デバイスを管理する顧客を選択からします。

2. 顧客の詳細 ページで、次のように選択します。**デバイス**します。

3. **プロファイルをデバイスに適用**選択**デバイスの追加**します。

4. デバイスの一覧の名前を入力し、**参照**(.csv ファイル形式) で、顧客の一覧をパートナー センターにアップロードします。

    >[!NOTE]
    >デバイス購入時にこの .csv ファイルを受信する必要があります。 .Csv ファイルを受信していない場合は、次の手順で作成する自分でできます[に Windows Autopilot デバイスを追加](https://docs.microsoft.com/windows/deployment/windows-autopilot/add-devices#collecting-the-hardware-id-from-existing-devices-using-powershell)します。  

5. .Csv ファイルをアップロードし、**保存**します。

.Csv ファイルをアップロードするときにエラー メッセージを取得する場合は、ファイルの形式を確認します。 ハードウェア ハッシュのみ、または OEM 名、シリアル番号、および (その列の順序で)、モデルまたは Windows プロダクト ID を使用することができます。 次に、リンクから提供されたサンプルの .csv ファイルを使用することもできます。**デバイスの追加**デバイス一覧を作成します。

## <a name="windows-autopilot-eula-dismissal"></a>Windows Autopilot の使用許諾契約書ジョンソン

### <a name="important-information"></a>重要な情報

Windows Autopilot の顧客を管理するデバイスで Windows のインストールをカスタマイズを構成することができます。 そのためには、顧客が承認されると場合、抑制または (エンド ユーザー ライセンス契約) 使用許諾契約書への同意の画面を含む、Windows をセットアップするときに、ユーザーに提示される通常設定の特定の画面を非表示にできます。

非表示にするか、ユーザーから、お客様の代わりに、用語、およびそのするを非表示にするための十分な同意と承認を取得した用語の意味の承認または通知を提供するように設計された任意の画面を非表示をこの関数を使用すると、同意します。(かどうか、組織または個々 のユーザーとして場合もあります)、お客様は、通知に同意して、いずれかをそのまま使用条項をお客様に適用されます。 これには、このツールによって画面を非表示にしない場合にユーザーに対して表示される、ライセンス契約条件や通知への同意を含みます。 お客様は、マイクロソフトまたはその認定代理店からソフトウェアのライセンスを正当に取得していない場合、デバイス上でその Windows ソフトウェアを使用することはできません。
