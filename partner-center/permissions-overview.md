# <a name="assign-users-roles-and-permissions"></a>ユーザー ロールとアクセス許可の割り当て


有効な名前とアドレス、サポートの詳細、税の除外対象をファイル、銀行情報、および会社の主要な連絡先を含む、パートナーのプロファイルを設定しました。 次の手順: をパートナー センターで作業を開始するため、パスワードとロールを設定するユーザーを取得します。

## <a name="set-up-your-employees-to-work-in-partner-center"></a>パートナー センターで使用する従業員を設定します。

役割とアクセス許可を付けると、パートナー センターにユーザーが持つアクセス権の種類を決定します。 ロールで、ビジネスが関係しているプログラムに関連しています。 たとえば、会社がクラウド ソリューション プロバイダー (CSP) ビジネスの場合は、必要はありませんのみ標準の Azure AD テナント グローバル管理者などの管理ロールが必要がありますの役割、CSP プログラムに固有です。 各プログラムには、固有の役割があります。

>[!Note]
> Azure Active Directory (AAD) テナント ロールには、グローバル管理者、ユーザー管理者、および CSP の役割が含まれます。 非 AAD のロールには、MPN 管理者、ビジネス プロファイルの管理、参照の管理、インセンティブの管理者およびインセンティブのユーザーが含まれます。
