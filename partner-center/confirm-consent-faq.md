## <a href="" id="processfaq"></a>プロセス

**Microsoft Cloud 契約に関するお客様の同意を確認するための新しいプロセスとは何ですか?**

マイクロソフトは、お客様とパートナーの契約に準拠した状態が維持されるよう支援することに取り組んでいます。 マイクロソフトのクラウド ソリューション プロバイダー (CSP) プログラムのパートナーは、マイクロソフトによる契約条件 (マイクロソフト製品およびサービスの利用規約を含む) を顧客との間で確認する責任があります。 パートナーがコンプライアンス要件を満たすのを支援し、透明性を確保するため、顧客が Microsoft Cloud 契約の条項に同意したことを確認することを CSP パートナーに要求する新しいプロセスを有効にしています。 2018 年 8 月 7日以降、パートナー センター ダッシュボードのユーザー インターフェイスとパートナー センター API に、パートナーがこの情報を提供するための新しいフィールドが導入されています。 この確認を提供することは最初は省略可能ですが、2018 年 11 月 7 日以降は必須です。 

**この要件は、いつ有効になりますか?**

パートナーは、2018 年 8 月 7日にこの機能が利用可能になると同時に、同意の確認を開始できます。 2018 年 11 月 7 日以降、パートナー センター ダッシュボードのユーザーは、すべての新規取引において、Microsoft Cloud 契約に顧客が同意することを確認する必要があります。 パートナー センター API のユーザーには、2019 年 3 月 22 日以降に同じことが要求されます。 API ユーザーの期限は、技術的な実装の余裕があるように延長されました。

**新規 | Microsoft は、パートナーによって提供された顧客情報が正しいことを検証しますか?**

Microsoft は、提供された顧客のメール アドレスが正しい形式であることを検証します。 正確な情報を提供し、不正確であることがわかったすべての情報を修正することは、パートナーの責任です。 

**新規 | パートナーは Microsoft Cloud 契約に対する顧客の同意をどのような方法で証明できますか (署名、文書、デジタル化、紙など)?**

パートナーには、顧客が Microsoft Cloud 契約の条項に同意したことを保証し、独自のプロセスとレコードを管理する責任があります。 パートナー センター ダッシュボードまたはパートナー センター API のプロセスを使用することにより、パートナーは顧客が Microsoft Cloud 契約の条項に同意したことを確認します。

**新規 | 顧客に対して確認プロセスが完了したことを、Microsoft はどのようにしてチェックしますか?**

2018 年 11 月 7 日以降、CSP パートナーがパートナー センター ダッシュボードを使用して新規または既存の顧客の取引を開始する場合、Microsoft のシステムにより、顧客の同意に関する確認情報がパートナー センター ダッシュボードに存在するかどうかのチェックが自動的に行われます。 存在しない場合は、取引を完了することができません。 

2019 年 3 月 22 日以降は、CSP パートナーがパートナー センター API を使用して新規または既存の顧客の取引を開始する場合、Microsoft のシステムにより、顧客の同意に関する確認情報が存在するかどうかのチェックが自動的に行われます。 存在しない場合は、取引を完了することができません。 

**新規 | 確認プロセスでメール アドレスが提供された顧客に通知が送信されますか?**

通知プロセスはありません。

## <a href="" id="policyfaq"></a>ポリシー

**顧客が Microsoft Cloud 契約に同意したことを確認するよう Microsoft がパートナーに要求するのはなぜですか?** 

Microsoft Cloud 契約は、Microsoft Cloud 製品およびサービスの注文に関して許可されている顧客の使用権や使用条件を提供することによって、パートナー、顧客、マイクロソフトを保護します。 

Microsoft Cloud 契約を適切に実行すると、セキュリティ、プライバシー、データ保護などの多くの重要なトピックに関して相互に一致することにより、パートナー、顧客、Microsoft が保護されます。 確認プロセスにより透明性と一致が保証され、規制に関する調査に速やかに応答する必要がある顧客を、Microsoft はいっそう適切に支援できます。

**確認プロセスを完了するためにどのような顧客情報が必要ですか?**
契約書に同意したユーザーに関して、次の情報が必要です。
-   契約に同意した日付
-   氏名
-   メール アドレス
-   電話番号 (オプション) 

**新規 | 期限を守らないとどうなりますか?**

パートナーは、2018 年 8 月 7日にこの機能が利用可能になると同時に、同意の確認を開始できます。 2018 年 11 月 7 日以降、パートナー センター ダッシュボードのユーザーは、すべての新規取引において、Microsoft Cloud 契約に顧客が同意することを確認する必要があります。 パートナー センター API のユーザーには、2019 年 3 月 22 日以降に同じことが要求されます。

適用後は、顧客の同意が確認されるまで、既存の顧客について、新しい購入や既存のサブスクリプションのシート数の変更は許可されません。 新しいお客様の場合、同意が確認されるまでは、取引は許可されません。 この要件は、自動更新には影響しません。

**この要件はどの地域に適用されますか?**

Microsoft Cloud 契約に対する顧客の同意の確認は、すべての地域について、Microsoft Public Cloud の下で必要になります。 お客様の同意の確認は、ナショナル クラウド (Microsoft Cloud for US Government や Microsoft Cloud Germany など) ではサポートされていません。

**Microsoft Cloud 契約に対する顧客の同意の確認は、どのパートナーが責任を負いますか?**

2018 年 11 月 7 日以降、クラウド サービス プロバイダー (CSP) 間接プロバイダーと CSP 直接請求パートナーの両方が、顧客の同意を確認する責任を負い、そのためにパートナー センター API またはパートナー センター ダッシュボードを使用できます。 間接プロバイダーは、まず間接リセラーから必要な情報を収集し、パートナー センター API またはパートナー センター ダッシュボードを通じて、マイクロソフトが使用できるようにする必要があります。 契約上、マイクロソフトと拘束力のある契約を結び、CSP リセラーのエンド ユーザーが Microsoft Cloud 契約に同意したことを確認するのは間接プロバイダーです。 

**Microsoft Cloud 契約に関する顧客の同意の確認が必要になるのは、どのような顧客シナリオですか?**

CSP パートナーが次の取引を行おうとする場合、Microsoft Cloud 契約に同意を確認する必要があります。
1.  新しい顧客が新規購入を完了する
2.  既存の顧客が新規購入を完了する
3.  既存の顧客が既存のサブスクリプションのシート数を更新する

**既存の顧客について、新規購入やシート数の追加とは別に同意を確認できますか?**

[はい]。 Microsoft Cloud 契約でカバーされるすべての顧客について、顧客の同意を確認できます。

**複数の CSP パートナーを持つ顧客のための要件は何ですか?**

同意の確認は、パートナーごとに行われます。 お客様が複数の CSP パートナーと取引している場合、各パートナーが同意を確認する必要があります。

**私は、CSP 間接リセラーと取引している CSP 間接プロバイダーです。私にはどのような責任がありますか?**

契約上、Microsoft と拘束力のある契約を結び、エンド ユーザーが Microsoft Cloud 契約に同意したことを確認するのは間接プロバイダーです。 ただし、間接プロバイダーは、お客様の同意を収集するために間接リセラーを別に配置している場合があります。 この場合、間接プロバイダーは、まず間接リセラーから必要なお客様の同意に関する情報を収集し、パートナー センター API またはパートナー センター ダッシュボードを通じて、マイクロソフトが使用できるようにする必要があります。 要求に応じて、速やかに間接プロバイダーに必要な情報を提供することで、このプロセスをサポートすることができます。

**私は CSP 間接プロバイダーです。CSP 間接リセラーと連携して準備する必要がありますか?**

間接リセラーから必要なお客様の同意情報を収集する必要がある間接プロバイダーは、この要件を間接リセラーに伝え、この情報の配信と受領を直ちに調整することをお勧めします。

**Microsoft Cloud 契約への顧客の同意を確認する必要があることを、顧客にどのように説明すればよいですか?**

Microsoft Cloud 契約を適切に実行すると、セキュリティ、プライバシー、データ保護などの多くの重要なトピックに関して相互に一致することにより、Microsoft、パートナー、顧客が保護されます。 確認プロセスにより透明性と一致が保証され、規制に関する調査に速やかに応答する必要がある顧客を、Microsoft はいっそう適切に支援できます。

**新規 | 顧客が 1 つの CSP 間接プロバイダーを利用していても、取引は複数の CSP 間接リセラーとの間で行う場合、各 CSP 間接リセラーが CSP 間接プロバイダーに確認情報を提供する必要がありますか?**

CSP 間接プロバイダーは、Microsoft Cloud 契約に対する顧客の同意の確認を、Microsoft に提供する責任があります。 パートナー センター ダッシュボードで一度に記録できる確認は、顧客アカウントごとに 1 つだけです。 

CSP 間接プロバイダーは、取引に関わった CSP 間接リセラーの数に関係なく、Microsoft Cloud 契約に同意する責任がある顧客担当者が同意したことを、確認する責任があります。

**新規 | 顧客の組織内で Microsoft Cloud 契約への同意を確認することを認められるのは誰ですか?**

顧客の組織内の人から Microsoft Cloud 契約に対する顧客の同意を得るのは、パートナーの責任です。 同意を提供することが認められる人を決定するのは、顧客の責任になります。 

**新規 | パートナーが顧客の同意を確認する責任があるのは、どのバージョンの Microsoft Cloud 契約ですか? パートナーは、顧客が同意した Microsoft Cloud 契約のバージョンを示す必要がありますか?**

パートナーは、新しい顧客と CSP の取引を行うとき、最新の Microsoft Cloud 契約を使用する責任があります。 パートナーは、既存の顧客に関する更新、変更、または新規取引の完了時に、最新の Microsoft Cloud 契約の同意を確認する必要があります。 

更改の前に Microsoft Cloud 契約が更新された場合、パートナーは、Microsoft Cloud 契約の顧客による同意をすぐに取り直す必要はありませんが、更改の前に Microsoft Cloud 契約への同意を再確認する必要があります。 ただし、Microsoft Cloud 契約の新しい改善された条項を利用するために更改する前に、顧客に新しい条項に同意してもらうことができます。

**新規 | 私は CSP 間接リセラーです。Microsoft Cloud 契約に対する顧客の同意を自社内に記録した場合、この情報をパートナー センター ダッシュボードまたはパートナー センター API でも提供する必要がありますか?**

契約上、Microsoft と拘束力のある契約を結び、エンド ユーザーが Microsoft Cloud 契約に同意したことを確認するのは間接プロバイダーです。 ただし、間接プロバイダーは、お客様の同意を収集するために間接リセラーを別に配置している場合があります。 この場合、間接プロバイダーは、まず間接リセラーから必要なお客様の同意に関する情報を収集し、パートナー センター API またはパートナー センター ダッシュボードを通じて、マイクロソフトが使用できるようにする必要があります。

**新規 | 顧客が Microsoft Cloud 契約の条項に同意しない場合はどうしますか?**

Microsoft の CSP プログラムのパートナーには、注文を行う前に、Microsoft Cloud 契約に対する顧客の同意を確認する責任があります。 これは新しい要件ではありません。 何らかの理由で、顧客が Microsoft Cloud 契約に同意しなくなった場合、パートナーはこの顧客に関連付けられているサブスクリプションを取り消す必要があります。

**新規 | リセラーは、顧客に代わって同意を提供できますか?**

いいえ。

**同意を確認した顧客に対し、後で再度確認する必要はありますか?**

顧客がいったん Microsoft Cloud 契約に同意したら、この情報はパートナー センターの顧客アカウント セクションに保存されるため、顧客がその契約条項の対象になっている期間中は、同意を再確認する必要はありません。 新しい注文を行う前にのみ、同意を再確認する必要があります。

更改の前に Microsoft Cloud 契約が更新された場合、パートナーは、Microsoft Cloud 契約の顧客による同意をすぐに取り直す必要はありませんが、更改の前に Microsoft Cloud 契約への同意を再確認する必要があります。 ただし、Microsoft Cloud 契約の新しい改善された条項を利用するために更改する前に、顧客に新しい条項に同意してもらうことができます。

**新規 | エンド カスタマーの直接の連絡先情報を Microsoft に提供するのは気が進みません。Microsoft はこの情報をどのように使うのですか?**

パートナーが顧客のテナントを作成するときは、顧客の主要な連絡先情報を提供する必要があります。 オンライン サービス使用条件にあるとおり、"Microsoft は顧客から書面での指示があったときにのみ個人データを処理します"。

## <a href="" id="technicalfaq"></a>技術

**パートナーはどのようにして顧客の同意を確認し、それを Microsoft に提供すればよいですか? このプロセスをサポートするにはどのツールを使用できますか?**

2018 年 8 月 7日以降、パートナーは 2 つの方法を使用できます。 
1.  パートナーは、パートナー センター API を使用して、プログラムによってお客様の同意を確認できます。 
2.  パートナーは、パートナー センター ダッシュボードを使用して、顧客の同意を確認できます。

**新規 | パートナー センター API を使用している CSP パートナーです。どのような API の変更を考慮する必要がありますか?** 

この場合、考慮する必要のある API の変更は次のとおりです。

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**パートナー センター API の実装に問題がある場合はどうしますか?**

パートナー センター ダッシュボードを使用して同意を手動で確認できます。

**新規 | パートナー センター ダッシュボードまたはパートナー センター API を使用する代わりに、顧客の確認情報が含まれるファイルを一括アップロードしてもかまいませんか?**

現時点では、Microsoft Cloud 契約に対する顧客の同意の確認を提供するための手段として受け入れられるのは、パートナー センター ダッシュボードとパートナー センター API だけです。

**新規 | 会社のシステムをパートナー センターと接続する機能を実行するサードパーティ製プラットフォーム (つまり "ミドルウェア") を使用しているパートナーです。何をする必要がありますか?**

パートナー センター API を統合するプラットフォームを使用しているパートナーは、顧客確認プロセスを自動化するパートナー センター API 機能を統合するための計画とタイムラインについて、プロバイダーと直接話すことをお勧めします。

**新規 | Microsoft Cloud 契約の顧客のフル ネーム、メール アドレス、日付をインポートするための API フローの例を提供してもらえますか?**

詳細と例については、パートナー センター API のドキュメントを確認することをお勧めします。

**新規 | 最新の Microsoft Cloud 契約を入手して顧客に示し、API プロセスの一部として確認記録で使用するにはどうすればよいですか?**

現在のパートナー センター API は非常に基本的なものであり、Microsoft Cloud 契約のコピーをプログラムで取得したり、確認の一部として顧客が同意した Microsoft Cloud 契約のバージョンをキャプチャしたりすることはできません。 

**新規 | このプロセスにパートナー センター API を使用するには、userID パラメーターが必要です。Azure AD と統合されていないサードパーティ製プラットフォームを通して注文が行われたときに、この要件を管理するにはどうすればよいですか?**

この場合の userID は、顧客の同意の確認を提供するパートナー テナントの下での従業員を示します。 従業員がパートナーのテナントでの対応する Azure AD ユーザー アカウントを持っていない場合は、その目的のために特定のユーザー アカウントを指定し、確認を提供する従業員の ID を別にキャプチャするプロセスを作成することをお勧めします。 

**新規 | Microsoft が、Microsoft Cloud 契約に対する顧客の同意の確認を、顧客から直接取得するのではなく、パートナーに要求するのはなぜですか?**

顧客は、Microsoft から直接購入するのではなく、CSP パートナーから購入します。 Microsoft が提供するサービスを使用する場合は、Microsoft Cloud 契約に記載されている条件を Microsoft が提供します。 Microsoft Cloud リセラー契約では、注文に先立って、CSP パートナーが顧客に Microsoft Cloud 契約を確認して同意してもらう必要があります。 

**新規 | Microsoft Cloud 契約に対する顧客の同意を確認するために、API とパートナー センター ダッシュボード Web ユーザー インターフェイスの両方を使っています。2018 年 11 月 7 日以降は、顧客の同意を確認するためにどのようなオプションがありますか?**

Microsoft Cloud 契約に対する顧客の同意を確認するには、どちらのオプションも使用可能で許容されます。 パートナー センター ダッシュボードを使用する取引では、2018 年 11 月 7 日以降に、確認の提示が必要になります。 パートナー センター API を使用する取引では、2019 年 3 月 22 日以降に、確認の提示が必要になります。
