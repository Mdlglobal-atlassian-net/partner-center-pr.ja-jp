# <a name="moving-from-pmc-to-partner-center"></a>PMC からパートナーセンターへの移行

Microsoft では、集中管理ポイントとして機能する単一の web サイト (パートナーセンター) を導入することで、ビジネスを容易にすることを実現しています。 パートナーメンバーシップセンター (PMC) で行ったことはすべて、パートナーセンターのダッシュボードから実行できます。 

また、1つの web サイトを離れることなく、はるかに多くのことを実現できます。 ただし、用語や機能によっては、見た目が異なる場合があります。 どこで何があるかについての最初の懸念を軽減するには、ダッシュボードのサイトツアーをご覧ください。

次の表は、PMC とパートナーセンター間の相違点の一部を示しています。

## <a name="renewing-your-microsoft-partner-network--membership"></a>Microsoft Partner Network のメンバーシップを更新しています

|**PMC**   |**パートナー センター**|
|----------------------|:-----------------------------|
|更新は記念日の前に90開始され、記念日までに完了している必要があります。| パートナーは、記念日から、最大30日後に更新を行うことができます。|

## <a name="account-structure"></a>アカウント構造

|**PMC**   |**パートナー センター**|
|----------------------|:-----------------------------|
|本社 + 場所–それぞれ個別に評価されます。 コンピテンシーの評価はローカルレベルで行われました|1人のグローバル企業 ("パートナーグローバルアカウント PGA" とも呼ばれます)。場所を含め、全体として評価されます。PGA レベルで集計されたパフォーマンスとスキルのデータパートナープロファイルや、紹介とマーケティング用のビジネスプロファイルなどのプログラム用のプロファイルビューがいくつか含まれています。|

## <a name="sign-in"></a>サインイン

|**PMC**   |**パートナー センター**|
|----------------------|:-----------------------------|
|Microsoft アカウント (MSA) または個人アカウントの資格情報を使用できるjoe@outlook.com|職場アカウントの資格情報 (joe@joescompany.com.) を使用する必要があります。|

## <a name="user-roles"></a>ユーザーロール

|**PMC**   |**パートナー センター**|
|----------------------|:-----------------------------|
|PMC に多数のロールがあり、パートナーセンターでは指定されていません。|パートナーセンターへの移動を実行する管理者には、MPN Admin、Account Admin、および参照管理者ロールが自動的に割り当てられます。 その後、ユーザーロールに他のユーザーを割り当てることができます。|
|ユーザーは場所レベルで管理されていました|ユーザーは、場所レベルではなく、会社レベル (PGA) で管理されます。 例外は、場所レベルで機能するインセンティブ管理者です。|
|   |パートナーセンターには、2つの広範なロールセットがあります。 Azure AD テナントを管理するロールと、会社のビジネスを管理するロールです。 会社にとって意味のある方法でロールを整理することができます。 だれでも、または多くの人が別のロールとアクセス許可を割り当てることができます。

|----------------------|:-----------------------------|

|2018年10月に提供が終了するまで、追加の特典ツールキット (ABTKs) が提供される可能性があります。|ありません。会社ごとに1つのマップ1社あたり1つのシルバーコンピテンシー会社ごとに1つの gold コンピテンシー|
|パートナーのデジタルダウンロード (PDD-2) で特典がアクセスされました |パートナーセンターですべての特典にアクセス|

|ソフトウェアアシュアランスの伝票請求は、バウチャー (VVR) ツールで作成されます。|パートナーセンター内で、ソフトウェアアシュアランストレーニングのバウチャー (SATV) や、展開された計画サービス (DPS) にアクセスして管理できるようになりました。  従来の VVR ツールは2019年10月1日に廃止される予定です。  |

可能。 Visual Studio の特典は、MSAs に割り当てられ、保持されます。 また、パートナーセンターでは更新後も保持されます。 ただし、パートナーセンターに移行した後に MSA 割り当てを削除しても、パートナーセンターに戻すことはできません。

パートナーセンターでは、パートナーが Azure AD テナントの MPN admin である同じテナントから、MSA である仕事用アカウントとゲストユーザーアカウントを追加できます。 パートナーが複数の Azure AD テナントのグローバル管理者であり、これらすべてのテナントが同じパートナーセンターアカウントに関連付けられている場合、パートナーは、これらすべてのテナントのユーザーを Visual Studio の特典と Azure の使用量ベースの割り当てに追加することができます。

ゲストユーザーは、MPN admin または全体管理者によって Visual Studio の使用量ベースのサブスクリプションを割り当てることができますが、ゲストユーザーは、MSA を使用してパートナーセンターにサインインすることはできません。 ただし、ゲストユーザーは、Azure と Visual Studio にサインインして、割り当てられた特典を検証して使用することができます。
